# PDF 표 병합 개선: 페이지 경계 병합 TODO

## 배경
현재 cross-page table merge는 pending table + 여러 휴리스틱을 조합해 진행되며, 현재는
a) 첫열 누락 특수케이스(3열→2열),
b) 텍스트 기반 gap 검사 중심 판정,
c) 페이지 내 모든 표를 병합 대상으로 순회하는 흐름이 혼재되어 있습니다.
요구사항에 맞춰 "다음 페이지 첫 표와만" 병합되고, body/table/text/image/note 기반 경계 관리에
기반한 더 명시적 병합 판정으로 정리합니다.

## 핵심 변경 방향 (1차)
1. 표/텍스트/이미지/노트의 페이지별 영역 관리 모델을 먼저 도입한다.
2. next-page 병합은 항상 “다음 페이지에서 본문의 첫 번째 표 후보”만 대상화한다.
3. 병합 판단에서 헤더 중복은 gap(빈 간격) 유무가 아니라 **표의 첫 행 비교**로 판단한다.
4. 3열→2열 특수 로직은 제거하고, 병합 자체는 구조/정렬/간격 일치 중심으로 통합한다.
5. 병합 후 정제(헤더 중복 제거, 텍스트 블록 정리)는 별도 후처리로 분리한다.
6. 축 정렬 tolerance는 현재 1.0으로 유지한다(1 차이내).
