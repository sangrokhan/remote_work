# 작업 운영 지침 (채팅 화면 UI/UX 프로젝트)

## 핵심 작업 흐름 (필수)

1. 사용자 요구사항 접수
2. 요청 내용을 `prd.md`에 반영 (요구사항, 구현 범위 또는 상태 항목)
3. 구현 수정 수행
4. 수정 완료 후 `prd.md`의 해당 항목을 검토/갱신
5. 배포
   - `docker compose up --build -d`
   - `docker compose ps`로 서비스 상태 확인

## 배포 전 필수 체크
- `prd.md`의 작업 항목이 최신 상태인지 확인
- 배포 전/후 상태를 PRD 완료 기준과 대조

## 배포 후
- `http://localhost:10000` 접속 가능 여부 확인
