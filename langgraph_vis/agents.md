# agents.md

## 개발 진행 가이드 (MVP/PoC 우선)

### 1) 개발 시작 원칙
- 코드 수정 전, 해당 주차의 `plan.md`, `context.md`, `checklist.md`를 먼저 읽고 의존성/완료 기준을 확인한다.
- 구현 범위는 이번 주차 계획의 `산출물`과 `완료 기준`에 한정한다.
- 주차 단위라 하더라도 작업은 작은 단위로 분해해 반복적으로 진행한다.
- 문서 검토 단계는 개발의 선행 조건이며, 문서 미검토 상태에서의 구현은 진행하지 않는다.

### 2) 1차 작업 흐름 (반드시 준수)
- 기본 개발 순서: `plan/context/checklist` 검토 → 검증에 필요한 테스트 케이스 도출 → 테스트 코드 선행 작성 → 구현 → 테스트.
- 문서 기반 목표(기능/산출물/완료 기준)와 테스트 항목이 1:1로 매칭되는지 확인한다.
- 테스트 우선 설계: 구현 전에 필요한 검증 테스트 케이스를 정리한다.
- 테스트 코드 선행 작성: 구현 전에 최소 단위 테스트/계약 테스트/통합 테스트부터 먼저 작성한다.
- 구현: 테스트가 실패하는 상태에서 단일 기능 단위로 구현한다.
- 즉시 검증: 작성한 테스트를 실행해 실패 원인을 고친 뒤 통과 상태를 확인한다.
- 반복: 다음 단위를 같은 방식으로 이어서 수행한다.

### 3) 최소 단위 반복 기준
- 각 반복은 1개 산출물 또는 1개 체크리스트 항목을 목표로 한다.
- “한 번에 끝내기”보다 “작게 쪼개어 실패/성공을 빠르게 확인”한다.
- 반복 종료 조건은 테스트 통과 + 산출물 체크리스트 갱신이다.

### 4) 테스트 우선 개발 체크
- 단위 테스트: DTO/스키마/변환기/파서/상태 머신의 작은 동작을 우선 검증한다.
- 통합 테스트: API 계약, 재동기화, reducer 반영, SSE 토픽별 흐름을 분리해 작성한다.
- 계약 검증: plan/context에서 정의한 필드, 코드/메시지/상태 규칙이 문서와 일치하는지 테스트로 고정한다.
- 리스크 테스트: 실패/재시도/연결 끊김/누락 이벤트 경로를 미리 넣어야 한다.

### 5) 주차별 수행 방식(권장)
- 1주차: API 계약과 고정 데이터부터, 그 다음 라우팅/검증 테스트.
- 2주차: run/state/mode 제어 등 상태 변화 단위를 먼저 고립 테스트 후 연결.
- 3주차: canonical 변환기와 history API를 작은 데이터셋으로 1:1 비교 테스트.
- 4주차: UI 렌더 파이프라인을 PoC 크기에서 검증하고, 성능/KPI 체크 항목은 기록 가능한 형태로 남긴다.

### 6) 검토/종결 규칙
- 모든 코드 변경은 checklist의 테스트 항목과 매핑한다.
- 기능 완료는 코드 + 테스트 + 문서(산출물) 3박자 모두 충족 시 판단한다.
- 구현 완료 전, 다음 순서에서 재사용할 테스트 시나리오와 결과를 문서화한다.
- 체크리스트 갱신은 테스트 성공 시에만 수행한다.
- 체크리스트 항목은 해당 주차의 `Subagent`에게 다음 4개 항목(개발문서 준수, 목표 작업 매칭, 테스트 커버리지, 실행 결과)을 종합 판단받아 하나씩 승인/체크한다.

#### 체크리스트 갱신용 Subagent 프롬프트 템플릿 (매 항목마다 사용)
- 역할: 구현 검증 담당
- 과제:
  1. `week-XX/plan.md`, `week-XX/context.md`, `week-XX/checklist.md`를 검토한다.
  2. 해당 체크리스트 항목의 요구사항과 개발 목표를 비교한다.
  3. 개발 코드 변경 파일, 테스트 코드 파일, 실행된 테스트 결과를 근거로 판단한다.
  4. 아래 형식으로 판단해 승인/거부를 명시한다.
    - 판단 기준: (a) 개발문서 반영 여부, (b) 목표 작업 내용 반영 여부, (c) 테스트 코드 존재 여부, (d) 실제 구현 반영 여부, (e) 테스트 통과 여부
    - 출력 형식:
      - 항목명:
      - 판단(승인/보류):
      - 근거: 개발문서 항목, 코드 경로, 테스트 경로/명령, 결과
      - 보류 사유(필요 시):
  5. 모든 항목이 승인되면 체크박스에 `[x]`를 반영한다.

### 2-회차 작업 플로우(체크리스트 포함)
- 테스트 실행 후 “실패 없음(Pass)”이 확인된 항목만 체크 대상으로 지정한다.
- 주차별 체크리스트 각 항목에 대해 Subagent 출력을 먼저 받고, 그 결과를 바탕으로 체크박스를 갱신한다.
- Subagent가 보류 판정한 항목은 즉시 다음 반복 단위로 분해해 보완 후 동일 절차를 재실행한다.

### 7) 작업 로그 규칙
- 변경 전후로 파일 경로, 체크리스트 항목, 테스트 결과를 간단히 기록한다.
- 미완료 항목은 다음 반복으로 분해해 배치한다.
